# 8. 延迟至设计阶段的议题

在多轮需求讨论中，以下议题被用户明确指为**设计阶段事项**或属于实现细节，本需求文档不下结论，仅记录在案，避免遗失。这些议题必须在系统设计阶段逐一回答。

## 8.1 工具系统

- **工具最小完备集合的具体清单**：除已确定必须存在的 `write_plan` 与 `task` 之外，文件类、搜索类、执行类、网络类工具的最终列表与每个工具的优先级（P0 / P1 / 不做）。
- **是否在 `write_plan` 之外再提供 `list_plan` 等辅助工具**。
- **每个工具的输入参数、输出结构、错误语义**。
- **权限模式与工具的对应关系**：在 `--plan`、默认、`--auto-edit`、`--yes` 四种模式下，每个工具是被禁用、需确认还是免确认。
- **`task` 工具调用子 agent 时的传参形式与返回结构**。

## 8.2 LLM Provider 与重试

- LLM Provider 抽象接口形态（方法签名、流式契约、function calling 适配方式）。
- 多 Provider 的注册与发现机制。
- 网络层重试与业务层重试的具体实现策略。
- usage 字段缺失时的回退策略。

## 8.3 数据存储

- SQLite 表结构设计（sessions、messages、todos、统计等表的字段、索引、关系）。
- 数据库文件默认位置与多数据库支持。
- 会话 ID、消息 ID 的生成方式。
- 持久化频次（每条消息落盘 vs 批量落盘）。

## 8.4 配置与项目约定

- `~/.mini-agent/config.yaml` 的最终字段结构。
- `AGENTS.md` 的查找路径策略（仅 cwd / 向上递归到 git root / 是否支持全局 `~/.mini-agent/AGENTS.md`）。
- 全局与项目级 `AGENTS.md` 的合并策略。
- `AGENTS.md` 的加载时机（启动时 / `/cd` 后 / `/reload` 命令）。
- `AGENTS.md` 注入到模型上下文的位置（系统消息拼接 vs 独立 system message）。
- 白黑名单规则文件的路径与格式。

## 8.5 上下文压缩

- `Compactor` 接口的具体形态。
- 摘要式、滑动窗口式、分层式策略的具体算法。
- 压缩触发的 token 阈值默认值与可配置范围。
- "保留最近 N 轮"中 N 的默认值。
- 压缩对工具消息、tool_call 关联结构的处理方式。

## 8.6 子 Agent

- `task` 工具的入参与子 agent 上下文初始化方式。
- 子 agent 在 Trace 中如何体现父子关系（trace_id、span_id、缩进展示等）。
- 子 agent 与主 agent 共享 todo 的具体读写边界（避免并发覆盖等问题）。

## 8.7 CLI / REPL

- 启动参数的具体名称与短选项。
- 斜杠命令的最终命名、缩写与帮助文案。
- 一次性 prompt 模式（`-p` / `--print`）的输入输出格式。
- Trace 实时显示的输出格式与开关粒度。

## 8.8 Web UI

- 后端 API 的协议选择（REST / SSE / WebSocket / 组合）。
- 前端项目工程结构与构建方式。
- 嵌入式静态文件分发 vs 独立部署。
- 前端的状态管理、组件库选型。
- 流式消息渲染、diff / markdown 渲染的实现方案。
- 权限审批卡片的交互细节（按钮、超时、批量批准）。

## 8.9 日志与 Trace

- 日志文件位置（用户级 vs 项目级 vs 数据库目录边）。
- 日志格式（JSON Lines vs 人类可读文本）。
- 日志级别体系。
- 日志轮转策略（按大小 / 按日期 / 保留天数）。
- 是否在 SQLite 中再冗余存储 traces 表（已倾向不存）。

## 8.10 项目结构

- Go 项目目录布局（`cmd/` + `internal/` 标准布局 / DDD 分层 / 平铺）。
- 模块划分与包命名。
- 依赖注入方式。

## 8.11 MCP Client

- MCP Server 的注册与发现机制。
- stdio / SSE / Streamable HTTP 三类传输的统一抽象。
- MCP 工具与内置工具的命名冲突处理。
- MCP 调用错误与超时处理。

## 8.12 MVP 范围最终确认

- 阶段划分中每阶段的最终功能清单，特别是 P0 包含的工具清单与压缩策略。
