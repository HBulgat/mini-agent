# 3. 非功能性需求

本章只描述**用户可感知**的非功能性约束，不涉及实现层指标。

## 3.1 性能与延迟

- 模型流式输出必须是**真正的增量推送**，用户应在模型生成第一个 token 后即可看到内容，而非等到整条响应完成。
- LLM 请求默认超时 60 秒，超时后明确告知用户失败原因。
- CLI REPL 在用户输入后必须**及时响应**（无肉眼可感的卡顿）；长任务应通过流式 / Trace 输出持续提供进度反馈。

## 3.2 可靠性与恢复

- LLM 网络层失败：等间隔重试 3 次后，向用户报告失败。
- 工具调用失败：在同一逻辑步骤内由模型重试至多 3 次后，提示模型换方式。
- Agent 循环达最大步数后必须**优雅停止**并汇报当前进展，不得静默挂起。
- 进程崩溃或被杀死后，已持久化的会话与 todo 必须可通过 `/resume` 恢复（不要求恢复"正在进行中"的循环状态，只要求历史消息可追溯）。

## 3.3 持久化

- 会话、todo 必须跨进程重启保留。
- 持久化介质为 SQLite，存放路径可由用户配置。
- 不强制在每次消息后立即落盘，但必须保证正常退出（含 `/exit` 与第二次 Ctrl+C）后数据完整。

## 3.4 可观测性

- 详细 Trace 是**强制要求**（不是可选项），覆盖范围见 §2.11。
- 用户必须能通过日志文件离线复盘整个 agent 行为。
- Trace 在 CLI 与 Web UI 都必须有可视化入口。

## 3.5 安全与隐私

- API Key 存储在用户本地配置文件中，**不得**输出到任何日志、Trace、CLI 显示或 Web UI 面板中。
- 内置硬黑名单（详见 `04-permissions.md`）即使在 `--yes` 模式下也必须拦截。
- 所有 shell 执行、文件写、文件删类工具必须经过权限模式判定，不得绕过。

## 3.6 配置可调试性

- 配置文件路径必须支持命令行参数覆盖，便于在开发期切换多套配置而不污染默认配置。

## 3.7 并发与资源使用

- 单个 mini-agent 进程在同一时刻**至多服务一个主任务循环**（不要求并发处理多任务）。
- 子 agent 调用为同步阻塞，因此不需要并发协调。
- Web UI 后端可同时服务多个浏览器标签的查询请求，但写入侧（创建会话、新增消息）的并发处理需求**不在 MVP 范围**，留待后续阶段评估。
