# 5. 扩展性需求

本章列出 mini-agent **必须支持扩展的子系统**及其扩展受众。具体扩展接口形式留待设计阶段确定。

## 5.1 LLM Provider

- **可扩展**：可加入新的协议族（OpenAI 兼容 / Anthropic / Gemini 之外的协议）。
- **扩展受众**：内部开发者（通过实现内置抽象、重新编译）。
- **运行时切换**：用户可在配置或 `/model` 命令中切换激活的 provider。

## 5.2 上下文压缩策略（Compactor）

- **可扩展**：内置策略至少包含摘要式、滑动窗口式、分层摘要式三种。
- **配置切换**：用户可通过配置选择启用何种内置策略。
- **扩展受众**：内部开发者；不要求支持运行时插件 / WASM / 外挂二进制。

## 5.3 工具集

- **不要求**支持运行时新增工具的插件机制（MCP client 之外）。
- 内置工具集合通过设计阶段固化，由内部开发者维护。

## 5.4 MCP Client

- **必须支持**通过 MCP 协议接入外部工具来源（P3 阶段）。
- 传输方式必须覆盖 stdio、SSE、Streamable HTTP 三类。
- 是面向**最终用户**的扩展点：用户可在配置中声明 MCP Server，运行时由 mini-agent 加载。

## 5.5 项目级 / 全局上下文

- 通过 `AGENTS.md` 文件注入项目级上下文，是面向最终用户的可扩展机制。
- 具体查找路径、合并策略、加载时机留待设计阶段。

## 5.6 Skill 加载机制

- 通过将 skill 目录放置在用户级（`~/.mini-agent/skills/<name>/`）或项目级（`<cwd>/.mini-agent/skills/<name>/`）目录下，**最终用户**可向 mini-agent 注入新的领域知识包。
- Skill 是面向最终用户的可扩展机制，但其本质是**模型可读取的领域知识包 + 受 `bash` 权限管控的脚本**，**不**等同于运行时代码插件，不允许定义新工具、新子 agent 或新斜杠命令。
- 详细规范见 `11-skill.md`。

## 5.7 不在扩展范围内

- 子 agent 的角色化预定义机制
- 跨会话长期记忆的扩展点
- UI 主题 / 渲染器扩展机制
- 命令 / 斜杠命令的用户级扩展机制（用户不可自定义新斜杠命令）
- 通过 skill 定义新工具、新子 agent、新斜杠命令
