# 1. 项目背景与产品形态

## 1.1 背景与目标

mini-agent 是一个对标 Claude Code 的智能编程 Agent，旨在为开发者提供基于大语言模型的多轮对话编程辅助能力。项目同时承担**学习目的**（理解 agent 工作原理）与**实际使用目的**（可在日常开发中替代或辅助同类工具），属于二者兼顾的定位。

**总体时间预算**：一周内交付 MVP。

## 1.2 产品形态

mini-agent 同时提供两种使用形态，二者共享同一套底层能力与持久化数据：

1. **CLI 工具**：交互式 REPL 形态，是主要的使用入口。
2. **Web UI**：基于浏览器的图形化使用界面，前后端分离架构，前端使用 React。

两种形态对同一个工作目录、同一份会话数据、同一份配置进行操作；用户在 CLI 中产生的会话数据可以在 Web UI 中查看与延续，反之亦然。

## 1.3 目标用户与角色

| 角色 | 描述 | 主要使用形态 |
|---|---|---|
| 个人开发者 | 在终端或浏览器中借助 mini-agent 完成编码、阅读代码、执行命令、修改文件等任务 | CLI 主，Web UI 辅 |
| 学习者 | 通过本项目源码学习 agent loop、工具调用、上下文压缩、子 agent 等核心机制 | 不限 |

本产品**不针对多用户、多租户、企业协作场景**，相关需求明确不在范围内（详见 `07-out-of-scope.md`）。

## 1.4 典型使用场景

### 场景 A：CLI 一次性任务

用户在工程目录下启动 mini-agent，输入一段自然语言描述（例如"读一下 main.go 然后帮我把日志接入加上 zap"），agent 通过工具调用读文件、修改文件、必要时执行命令，完成后回到 REPL 继续等待下一轮输入。

### 场景 B：CLI 多步任务并自主规划

用户提出较复杂的需求（例如"重构当前包的错误处理"），模型自主调用 `write_plan` 工具产出 todo 列表，然后逐项推进，期间可能触发子 agent。用户可通过斜杠命令查看 todo、切换权限模式、压缩上下文。

### 场景 C：Web UI 富交互

用户在浏览器中打开 Web UI，左侧选择会话或新建会话，中部观察模型流式输出与工具调用过程，右侧通过面板查看 todo、文件树、文件 diff、权限审批卡片、Trace 等。CLI 与 Web UI 共用同一会话数据。

### 场景 D：恢复历史会话

用户重新打开 mini-agent，使用 `/resume` 命令选择之前的会话继续工作，原有上下文（含已被压缩的摘要）被加载回当前对话。

## 1.5 时间约束

- 设计与需求分析：一次性产出完整方案。
- 实现：分阶段，按 `06-phasing.md` 的优先级推进。
- 总周期：一周内交付可用 MVP。
