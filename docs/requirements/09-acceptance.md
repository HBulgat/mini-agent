# 9. 验收标准

本章为 MVP 各项功能给出**用户可独立验证**的通过条件。每条验收项的检验方式都是：执行特定操作、观察可见结果。

## 9.1 LLM 接入

- **流式输出**：发送一条普通 prompt，能在模型生成第一个 token 后立即看到文本增量出现，而非等待全部生成完毕后一次性出现。
- **原生 Function Calling**：触发一次需调用工具的任务，工具调用通过模型 API 的 function calling 字段流转，而非由 mini-agent 自行解析自然语言。
- **多 provider 切换**：通过命令切换激活的 provider 后，新的 LLM 请求走切换后的协议（可在日志中确认目标 base_url / 协议）。
- **超时与重试**：人为制造 LLM 网络异常（断网或拦截），可观察到 mini-agent 在 60 秒后判定超时，并按等间隔策略最多重试 3 次后向用户报告失败。
- **Token 统计**：连续发送若干请求，通过 `/cost` 命令、Web UI 面板与日志三处都能看到一致的累计 token 与单次请求 token，统计来源于 LLM response 的 usage 字段。

## 9.2 Agent 循环

- **轻规划**：复杂任务下，模型自行调用 `write_plan` 工具产出 todo，可通过 `/todos` 命令或 Web UI 面板看到该 todo 列表的实时变化。
- **步数上限**：构造一个长循环任务并把步数上限设到很小值，能观察到 agent 达上限后停止并汇报当前进展。
- **工具失败重试**：人为构造一个会失败的工具调用，能观察到模型重试 3 次，第 4 次时收到"请尝试换一种方式"的提示信息。
- **Ctrl+C 中断**：长任务执行中按一次 Ctrl+C，任务停止并回到 REPL；连续两次 Ctrl+C 退出 REPL。

## 9.3 子 Agent

- **同步派发**：触发一个会调用 `task` 的任务，主 agent 在子 agent 完成前不继续推理。
- **嵌套限制**：尝试在子 agent 中再调用 `task`，能观察到该调用被拒绝。
- **Todo 互见**：子 agent 写入的 todo 在 `/todos` 视图中与主 agent 的 todo 共同呈现。
- **失败暴露**：子 agent 失败后，主 agent 的下一步 LLM 请求中能看到子 agent 的错误信息。

## 9.4 上下文压缩

- **手动压缩**：执行 `/compact` 后，会话当前消息数显著减少，但系统 Prompt、`AGENTS.md`、最近若干轮原文仍完整保留。
- **自动压缩**：构造长会话使 token 占用超过阈值，能观察到自动触发压缩，且日志中有压缩事件记录。

## 9.5 会话与持久化

- **新建与恢复**：启动 REPL 默认进入新会话；执行 `/sessions` 列出历史会话；`/resume` 选择某历史会话后，原对话上下文可见。
- **进程重启**：完成会话后退出 mini-agent，再次启动通过 `/resume` 恢复，会话内容、todo、统计计数完整。

## 9.6 权限模式

- **默认模式**：触发 shell 执行类工具时，CLI 弹出确认提示。
- **`--auto-edit`**：写文件免确认，但 shell 执行仍需确认。
- **`--yes`**：常规工具调用全部免确认；触发硬黑名单命令仍被拒绝。
- **`--plan`**：尝试触发写文件或 shell 执行被拒绝；只读类工具正常工作。
- **白黑名单**：在规则文件中加入命令级 / 路径级 / 工具级规则，可观察到对应行为变化。

## 9.7 配置

- 默认从 `~/.mini-agent/config.yaml` 加载；通过命令行参数指定其他路径时优先使用指定路径。
- 修改配置中的 model 字段后重启，激活的模型变更。

## 9.8 CLI / REPL

- 全部斜杠命令均能正常执行，输出符合各自语义。
- 一次性 prompt 模式（非交互）下，进程执行完成后退出，输出正确。

## 9.9 Web UI

- 浏览器打开 Web UI，可看到与 CLI 共享的会话列表。
- 在 Web UI 中创建会话、发送消息，CLI 重新启动后通过 `/sessions` 也能看到该会话。
- 模型流式输出在 Web UI 中实时渲染。
- 工具调用可视化展示包含输入、输出、耗时。
- 文件树、文件预览 / diff、Todo 面板、Trace 面板均可正常使用。
- 权限审批以对话框内卡片形式呈现，点击批准 / 拒绝后行为符合预期。
- 配置编辑界面可修改并保存配置。

## 9.10 Trace 与日志

- 日志文件中包含 LLM 请求、工具调用、Agent 决策、压缩事件、权限审批事件五类记录。
- `/trace on` 与 `/trace off` 切换 CLI 实时显示开关。
- 日志中**不出现** API Key 等敏感信息。

## 9.11 安全

- 在 `--yes` 模式下尝试触发硬黑名单命令，被明确拒绝，错误信息提示该命令被硬黑名单拦截。
- 任意展示界面（CLI、Web UI、日志）中均不包含 API Key 字面量。

## 9.12 Skill 模块（P1）

- **启动注入**：在用户级或项目级目录放置一个含合规 SKILL.md 的 skill 后启动 mini-agent，可在系统提示词中观察到该 skill 的 `name + description` 已被注入（可通过 trace 日志确认）。
- **模型自动加载**：构造一段会命中某 skill description 的用户输入，模型自动调用 `skill_tool` 加载该 skill，trace 日志中可见加载事件。
- **用户显式触发**：在 REPL 中执行 `/skill <name>`，下一轮模型可读到该 skill 的 SKILL.md 内容。
- **优先级**：在用户级与项目级同时放置同名 skill，加载时使用的是项目级版本。
- **脚本权限**：在 `--yes` 模式下，skill 脚本中触发硬黑名单的命令仍被拒绝；在默认模式下，skill 脚本通过 bash 执行时弹出确认。
- **子 agent 继承**：主 agent 加载 skill 后调用 `task` 派生子 agent，子 agent 的上下文中包含已加载的 skill 内容。
- **压缩后再注入**：触发 `/compact` 后，skill 列表（`name + description`）依然存在于系统提示词中。
- **Web UI（P2）**：Web UI 中可在 skill 详情面板看到当前可用 skill 列表，点击可查看 SKILL.md 内容预览。
