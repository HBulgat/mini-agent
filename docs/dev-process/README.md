# mini-agent 开发过程文档

本目录跟踪 mini-agent 项目的**开发执行**过程：开发计划、迭代分组、进度记录、风险登记。与上层文档的关系如下：

| 目录 | 作用 | 回答的问题 |
|---|---|---|
| `../requirements/` | 需求文档 | 做什么、不做什么、边界在哪、验收什么样 |
| `../system-design/` | 系统设计 | 怎么做（接口、schema、协议、算法） |
| `../dev-process/` | 开发执行 | **何时做、按什么顺序做、做到哪了、做完算完** |

## 使用纪律

1. **开发前必读 `01-development-plan.md`**：了解当前迭代的任务边界、依赖、DoD。
2. **开发后必更新 `02-progress.md`**：把刚完成的任务从"进行中"标记为"完成"，记录实际产出、问题、下一步。
3. **范围变更回到上游**：如发现需要新增功能或调整范围，**先回到需求层走"同意需求分析结论"流程**，再修改设计与计划。
4. **设计先行**：原则上每一项任务在开始之前，对应的系统设计内容必须已锁定。否则在进度表中标注"等待设计"。
5. **不在本目录写代码细节**：实现细节属于系统设计。本目录只关心"任务编号、状态、产出、阻塞"。
